# Start here

Command Deck is a practical map of the lab workflow. It is not a replacement for thinking, but it should reduce how often new students are expected to guess invisible conventions.

## What this onboarding pack is for

Use this pack to learn:

- how the lab uses Git, GitHub, and the terminal;
- how to choose between `uv`, `conda`, `mamba`, containers, and cloud machines;
- how to work with papers, references, Overleaf, and Quarto;
- how to start using lab-specific simulation tools such as Taichi and MOOSE.

## Recommended order

1. Read **Day 0 checklist**.
2. Read **Terminal basics**.
3. Read **Git basics** and **Safe GitHub auth**.
4. Read **uv vs conda** before making environments.
5. Read **Containers basics** before using Docker or Singularity.
6. Read **GCP cost and safety** before starting cloud VMs.
7. Read the writing/research guides before touching shared papers.

## The rule

When you do not understand a command, do not run it just because a card can assemble it. Open the guide, read the intent, then run the command.
